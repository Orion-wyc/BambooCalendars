# BambooCalendars (质日) - 前后端技术设计文档

## 1. 项目概述

### 1.1 项目信息
- **项目名称**: BambooCalendars
- **中文名称**: 质日
- **项目类型**: 前后端分离的 Todo 管理应用
- **部署方式**: 本地开发环境
- **访问方式**: Chrome 等现代浏览器

### 1.2 核心价值
- 简洁高效的 Todo 任务管理
- 支持附件和图片贴图
- 现代化的用户界面
- 本地数据存储，隐私安全

## 2. 技术栈选择

### 2.1 后端技术栈

#### 2.1.1 核心框架
- **Web 框架**: Flask 3.0+
  - 理由：轻量级、灵活、易于开发
- **Python 版本**: Python 3.9+
- **WSGI 服务器**: Gunicorn
  - 生产环境使用

#### 2.1.2 数据库
- **主数据库**: SQLite
  - 理由：无需额外安装，适合本地开发
- **ORM**: SQLAlchemy
  - 理由：Python 最流行的 ORM，支持多种数据库
- **数据库迁移**: Flask-Migrate (Alembic)

#### 2.1.3 API 设计
- **API 风格**: RESTful API
- **数据格式**: JSON
- **API 文档**: Flask-RESTX / Swagger
- **CORS**: Flask-CORS

#### 2.1.4 文件处理
- **文件上传**: Flask 内置支持
- **图片处理**: Pillow (PIL)
- **文件存储**: 本地文件系统
- **存储路径**: `./uploads/`

#### 2.1.5 认证与安全
- **认证方式**: JWT (JSON Web Token)
- **密码加密**: bcrypt
- **输入验证**: Marshmallow / Pydantic
- **安全防护**: Flask-Security

#### 2.1.6 开发工具
- **依赖管理**: pip / poetry
- **环境管理**: python-venv
- **代码格式化**: Black / isort
- **代码检查**: Flake8 / pylint
- **测试框架**: pytest

### 2.2 前端技术栈

#### 2.2.1 核心框架
- **框架**: React 18+
  - 理由：生态丰富、组件化、性能优异
- **构建工具**: Vite
  - 理由：快速、现代、开发体验好
- **语言**: TypeScript
  - 理由：类型安全、提高代码质量

#### 2.2.2 UI 框架
- **UI 库**: Ant Design 5.x
  - 理由：企业级 UI 组件库，设计现代
- **图标库**: @ant-design/icons
- **样式方案**: CSS Modules + Tailwind CSS
  - 理由：组件隔离 + 实用类

#### 2.2.3 状态管理
- **状态管理**: Zustand / Redux Toolkit
  - 理由：轻量级、简单易用
- **数据获取**: React Query (TanStack Query)

#### 2.2.4 路由管理

#### 2.2.5 表单处理
- **表单库**: React Hook Form
  - 理由：性能好、API 简洁
- **表单验证**: Zod / Yup

#### 2.2.6 文件处理
- **文件上传**: axios / fetch
- **图片预览**: react-image-gallery / lightbox

#### 2.2.7 开发工具

## 4. 数据库设计

### 4.1 数据表设计

#### 4.1.1 用户表 (users)
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.1.2 Todo 表 (todos)
```sql
CREATE TABLE todos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    is_completed BOOLEAN DEFAULT FALSE,
    priority VARCHAR(20) DEFAULT 'medium',
    due_date TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 4.1.3 附件表 (attachments)
```sql
CREATE TABLE attachments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    todo_id INTEGER NOT NULL,
    filename VARCHAR(255) NOT NULL,
    original_filename VARCHAR(255) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    file_size INTEGER,
    mime_type VARCHAR(100),
    file_type VARCHAR(50),  -- 'image', 'document', 'other'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE CASCADE
);
```

### 4.2 数据模型 (Python)

```python
# models/user.py
class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(50), unique=True, nullable=False)
    email = db.Column(db.String(100), unique=True, nullable=False)
    password_hash = db.Column(db.String(255), nullable=False)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow)
    todos = db.relationship('Todo', backref='user', lazy=True)

# models/todo.py
class Todo(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    user_id = db.Column(db.Integer, db.ForeignKey('users.id'), nullable=False)
    title = db.Column(db.String(200), nullable=False)
    description = db.Column(db.Text)
    is_completed = db.Column(db.Boolean, default=False)
    priority = db.Column(db.String(20), default='medium')
    due_date = db.Column(db.DateTime)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow)
    attachments = db.relationship('Attachment', backref='todo', 
                                  cascade='all, delete-orphan')

# models/attachment.py
class Attachment(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    todo_id = db.Column(db.Integer, db.ForeignKey('todos.id'), nullable=False)
    filename = db.Column(db.String(255), nullable=False)
    original_filename = db.Column(db.String(255), nullable=False)
    file_path = db.Column(db.String(500), nullable=False)
    file_size = db.Column(db.Integer)
    mime_type = db.Column(db.String(100))
    file_type = db.Column(db.String(50))
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
```

## 5. API 设计

### 5.1 API 端点设计

#### 5.1.1 认证相关
```
POST   /api/auth/register    # 用户注册
POST   /api/auth/login       # 用户登录
POST   /api/auth/logout      # 用户登出
GET    /api/auth/me          # 获取当前用户信息
```

#### 5.1.2 Todo 相关
```
GET    /api/todos            # 获取所有 Todo
GET    /api/todos/:id        # 获取单个 Todo
POST   /api/todos            # 创建 Todo
PUT    /api/todos/:id        # 更新 Todo
DELETE /api/todos/:id        # 删除 Todo
PATCH  /api/todos/:id/complete  # 标记完成
```

#### 5.1.3 附件相关
```
GET    /api/todos/:id/attachments        # 获取 Todo 的所有附件
POST   /api/todos/:id/attachments        # 上传附件
GET    /api/attachments/:id              # 获取附件详情
GET    /api/attachments/:id/download     # 下载附件
DELETE /api/attachments/:id              # 删除附件
POST   /api/attachments/:id/preview      # 生成预览
```

## 7. 文件处理方案

### 7.1 文件上传
- **上传方式**: FormData + axios
- **文件大小限制**: 10MB
- **支持格式**: 
  - 图片: jpg, jpeg, png, gif, webp
  - 文档: pdf, doc, docx, txt, md
  - 其他: zip, rar

### 7.2 文件存储
- **存储路径**: `./uploads/{year}/{month}/{user_id}/`
- **文件命名**: `{timestamp}_{uuid}_{original_name}`
- **目录结构**: 按年月和用户分目录

### 7.3 图片处理
- **缩略图生成**: 自动生成 200x200 缩略图
- **图片压缩**: 超过 2MB 自动压缩
- **格式转换**: 统一转换为 webp 格式

### 7.4 预览功能支持
- **图片**: 原生预览，支持缩放旋转
- **PDF**: 使用 PDF.js 预览
- **文本**: 直接显示内容
- **其他**: 显示文件信息
